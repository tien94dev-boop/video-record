Lý Thuyết Cơ Bản & Bài Tập Thực Hành

Nội dung bài giảng

  • Phần 1: Lý thuyết căn bản (SELECT, WHERE, JOIN)
  • Phần 2: Thiết lập dữ liệu mẫu (Câu 1)
  • Phần 3: Thực hành truy vấn dữ liệu (Câu 2 - Câu 10)
  • Phần 4: Thống kê & Kết nối bảng (Câu 11 - Câu 18)

Lý thuyết 1: Lệnh SELECT & WHERE

Đây là cấu trúc cơ bản nhất để lấy dữ liệu từ một bảng.

Cú pháp chính

SELECT cột1, cột2
FROM tên_bảng
WHERE điều_kiện;

Ví dụ thực tế

Lấy tên các sinh viên ở Yokohama:

SELECT name
FROM student
WHERE address = 'Yokohama';

Lý thuyết 2: Toán tử so sánh & Logic

Dùng trong mệnh đề WHERE để lọc dữ liệu chính xác hơn.

So sánh & Tìm kiếm

  • = , <> (Bằng , Khác)
  • BETWEEN 70 AND 80 (Trong khoảng)
  • LIKE 'A%' (Bắt đầu bằng A)
  • LIKE '__2%' (Ký tự thứ 3 là số 2)

Kết hợp Logic

  • AND: Thỏa mãn cả hai
  • OR: Thỏa mãn một trong hai
  • NOT: Phủ định điều kiện

Lý thuyết 3: Hàm gộp & Sắp xếp

Hàm thống kê (Aggregate)

  • SUM(): Tính tổng giá trị
  • AVG(): Tính trung bình
  • COUNT(): Đếm số bản ghi

* Thường đi kèm với GROUP BY

Sắp xếp (ORDER BY)

ORDER BY cột ASC; -- Tăng dần
ORDER BY cột DESC; -- Giảm dần

Lý thuyết 4: Kết nối bảng (JOIN)

Dùng khi cần lấy dữ liệu từ nhiều bảng khác nhau có liên kết với nhau.

SELECT s.name, sc.score
FROM student s
JOIN score sc ON s.student_number = sc.student_number;

Mẹo: Sử dụng Alias (tên thay thế như s, sc) giúp câu lệnh ngắn gọn và dễ đọc hơn.

Câu 1: Lệnh tạo bảng (DDL)

CREATE TABLE student (
    student_number CHAR(4) PRIMARY KEY,
    name VARCHAR(50),
    gender VARCHAR(10),
    address VARCHAR(100)
);

CREATE TABLE subject (
    subject_code CHAR(3) PRIMARY KEY,
    subject_name VARCHAR(50)
);

CREATE TABLE score (
    student_number CHAR(4),
    subject_code CHAR(3),
    score INT,
    examination_date DATE,
    PRIMARY KEY (student_number, subject_code)
);
